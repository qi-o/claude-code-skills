# Content Analysis (InkOS 集成版)

## 目标

- 使用 InkOS 内置分析能力提取结构化信息
- 支持 4 个分析维度：角色、世界观、伏笔/线索、章节摘要
- 输出可被 `/webnovel-write` 消费的结构化数据

## 前置条件

- InkOS 项目中有已写章节，或用户提供了分析内容

## 分析方式

### 方式 A：InkOS 项目分析（推荐）

当分析 InkOS 项目中的内容时，使用 CLI：

```bash
# 审查分析（37+维度质量报告）
inkos audit <book-id>

# 风格分析
inkos style <book-id>

# 项目状态（含 Truth Files 摘要）
inkos status <book-id>

# 读取 Truth Files 获取结构化数据
inkos read-truth <book-id>
```

InkOS 的 ChapterAnalyzerAgent 自动从每章中提取并更新：
- **角色**：出场角色、关系变化、位置追踪 → `character_matrix.md`
- **世界观**：地点、势力、能力体系 → `story_bible.md`
- **伏笔**：钩子、承诺、悬念 → `pending_hooks.md`
- **章节摘要**：核心事件、情感基调 → `chapter_summaries.md`

### 方式 B：自由文本分析

当分析非 InkOS 项目的文本时，使用 AI 直接分析。

输入类型：

| 输入类型 | 说明 | 处理方式 |
|----------|------|----------|
| 单个文件 | 一个章节文件 | 读取并分析 |
| 目录 | 多个章节 | 扫描全部并汇总 |
| 粘贴文本 | 用户直接提供内容 | 直接分析 |
| 项目目录 | 已有网文项目 | 读取所有章节 |

## 分析维度

### 1. 角色提取

从内容中提取：
- **角色名称**、**角色定位**（主角/配角/反派）
- **角色特征**（性格、身份）
- **角色关系网**

输出格式：
```json
{
  "characters": [
    {
      "name": "角色名",
      "role": "主角/配角/反派/女主",
      "traits": ["特征1", "特征2"],
      "relationships": [{"target": "相关角色", "type": "朋友/敌人/恋人"}]
    }
  ]
}
```

### 2. 世界观提取

从内容中提取：
- **地点**、**势力**、**能力体系**、**社会规则**

### 3. 伏笔/线索提取

从内容中提取：
- **悬念**（未解之谜）
- **承诺**（情节承诺）
- **未完成事件**
- **暗示**（隐含信息）

### 4. 章节摘要

为每个章节生成摘要：
- **核心事件**、**出场人物**、**情感基调**

## 交互流程

### Step 1：确定分析目标

1. 用户提供了 InkOS 项目？→ 使用方式 A
2. 用户提供了自由文本？→ 使用方式 B
3. 用户指定了分析维度？→ 只做指定维度

### Step 2：执行分析

**InkOS 项目**：
```bash
# 获取项目状态和 Truth Files
inkos status <book-id>

# 质量审查报告
inkos audit <book-id>

# 风格指纹
inkos style <book-id>
```

**自由文本**：
- 读取用户提供的内容
- 按分析维度提取结构化信息

### Step 3：输出结果

- **直接显示**：打印分析结果
- **写入文件**：保存到分析报告
- **更新 InkOS 项目**：写入 Truth Files

## 验证与交付

成功标准：
- 提供的内容已被分析
- 角色/世界观/伏笔已提取
- 摘要已生成
- 如为 InkOS 项目，Truth Files 已更新

## 后续操作

1. **开始续写**：`/webnovel-write`
2. **继续分析**：`/webnovel-analyze`（可重复分析新增章节）
3. **质量审查**：`inkos audit <book-id>`

## 用户确认检查点

以下操作前**必须暂停并询问用户确认**：

| 检查点 | 触发条件 | 确认内容 |
|--------|---------|---------|
| 分析维度选择 | 用户未指定具体维度 | 列出可用维度（角色/世界观/伏笔/摘要），确认需要哪些维度 |
| Truth Files 更新 | 分析结果将写入 InkOS 项目文件 | 确认写入目标，展示即将修改的文件列表，避免覆盖已有数据 |
| 自由文本范围 | 输入为目录或多文件 | 确认要分析的文件范围，排除不需要的文件（草稿、备份等） |

## 错误处理与回退

| 错误场景 | 检测信号 | 回退策略 |
|---------|---------|---------|
| InkOS CLI 不可用 | `inkos` 命令返回非零退出码 | 回退到方式 B（AI 自由文本分析），直接读取文件并用 AI 提取结构化信息 |
| 章节内容为空 | 读取的章节文件无实质内容 | 跳过空文件并记录，在最终报告中标注缺失章节 |
| 角色关系矛盾 | 提取的角色关系与已有 Truth Files 冲突 | 展示矛盾点，询问用户以哪个版本为准，手动修正后继续 |
