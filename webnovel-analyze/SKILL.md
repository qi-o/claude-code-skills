# Content Analysis (内容分析模块)

## 目标

- 分析用户提供的已有小说内容（章节/大纲/设定）
- 提取结构化信息：角色、世界观、伏笔/线索
- 生成可被 `/webnovel-write` 消费的结构化数据

## 执行原则

1. 支持多种输入形式：文件/目录/粘贴文本
2. 提取信息要准确，避免过度推断
3. 输出格式要便于后续写作时召回

## 输入类型

| 输入类型 | 说明 | 处理方式 |
|----------|------|----------|
| 单个文件 | 一个章节文件 | 读取并分析 |
| 目录 | 多个章节 | 扫描全部并汇总分析 |
| 粘贴文本 | 用户直接提供内容 | 直接分析 |
| 项目目录 | 已有网文项目 | 读取所有章节 |

## 分析维度

### 1. 角色提取

从内容中提取：

- **角色名称**：出现的所有人名
- **角色定位**：主角/配角/反派/龙套
- **角色特征**：性格、身份、关系
- **角色互动**：角色之间的关系网

输出格式：
```json
{
  "characters": [
    {
      "name": "角色名",
      "role": "主角/配角/反派/女主",
      "traits": ["特征1", "特征2"],
      "relationships": [
        {"target": "相关角色", "type": "朋友/敌人/恋人"}
      ]
    }
  ]
}
```

### 2. 世界观提取

从内容中提取：

- **地点**：故事发生的地点
- **势力**：宗门/组织/国家等
- **能力体系**：修炼等级/技能系统
- **社会规则**：货币/法律/风俗

输出格式：
```json
{
  "world": {
    "locations": ["地点1", "地点2"],
    "factions": [
      {"name": "势力名", "type": "宗门/国家/组织"}
    ],
    "power_system": "力量体系描述",
    "rules": ["规则1", "规则2"]
  }
}
```

### 3. 伏笔/线索提取

从内容中提取：

- **悬念**：未解之谜
- **承诺**：情节承诺（如"日后必报"）
- **未完成事件**：中断的情节线
- **暗示**：隐含的信息

输出格式：
```json
{
  "plot_threads": [
    {
      "type": "悬念/承诺/未完成/暗示",
      "description": "描述",
      "first_appeared": "章节号",
      "status": "open/resolved"
    }
  ]
}
```

### 4. 章节摘要

为每个章节生成摘要：

- **字数**：章节字数
- **核心事件**：本章主要发生了什么
- **出场人物**：本章出现的角色
- **情感基调**：本章的情绪走向

输出格式：
```json
{
  "chapters": [
    {
      "number": 1,
      "title": "章节标题",
      "word_count": 2000,
      "summary": "100字摘要",
      "characters": ["角色A", "角色B"],
      "mood": "紧张/轻松/温馨"
    }
  ]
}
```

## 交互流程

### Step 1：收集输入

询问用户提供分析内容：

1. **文件/目录路径**：已有章节所在位置
2. **粘贴内容**：直接提供要分析的文本
3. **项目导入**：指定要分析的项目目录

### Step 2：执行分析

1. **读取内容**：
   - 读取用户提供的一个或多个文件
   - 或扫描目录下的所有章节文件

2. **结构化提取**：
   - 使用 AI 能力提取角色/世界观/伏笔
   - 按上述格式组织数据

3. **生成摘要**：
   - 为每个章节生成100-200字摘要
   - 标记核心事件和出场人物

### Step 3：输出结果

输出结构化分析结果，可选：

1. **直接显示**：打印分析结果
2. **写入文件**：保存到分析报告
3. **更新项目**：写入 `.webnovel/` 供后续使用

### Step 4：可选 - 写入项目

若用户已有项目，可选择：

- 更新 `设定集/角色汇总.md`
- 更新 `设定集/伏笔追踪.md`
- 生成 `.webnovel/summaries/` 摘要

## 使用场景

### 场景1：导入后首次分析

用户刚用 `/webnovel-import` 导入已有内容，需要分析：

```
> 分析一下这个项目的角色和伏笔
```

### 场景2：写作前回顾

写新章节前，想了解之前的内容：

```
> 帮我回顾一下第5-10章的主要内容
```

### 场景3：检查一致性

检查新写章节与已有内容的一致性：

```
> 检查第20章和之前设定是否有冲突
```

## 验证与交付

成功标准：
- 提供的内容已被分析
- 角色/世界观/伏笔已提取
- 摘要已生成

输出示例：

```
=== 角色分析 ===
主角：张三
- 性格：沉稳、腹黑
- 关系：李四（朋友）、王五（敌人）

=== 世界观 ===
地点：青云城、玄天宗
势力：玄天宗（正道）、魔渊（反派）

=== 伏笔 ===
1. [第3章] 张三的玉佩来源（悬念，open）
2. [第5章] 承诺击败魔渊（承诺，open）
```

## 后续操作

分析完成后，提示用户：

1. **开始续写**：`/webnovel-write {next_chapter}`
2. **查看详细**：`/webnovel-query characters`
3. **继续分析**：`/webnovel-analyze`（可重复分析新增章节）
