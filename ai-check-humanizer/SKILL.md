# AI 检测与人性化改写技能

## 目的

本技能融合了 **AI检测** 与 **人性化改写** 两大功能：
1. **检测**：识别典型的 LLM 生成文本模式，评估AI痕迹程度
2. **改写**：提供详细的修复指南，让文本更自然、更有人味

> 核心理念：**不仅要干净，更要鲜活**

## 何时使用此技能

### 主要使用场景
1. **预提交验证** - 在 git 提交之前自动检查文档
2. **手稿审查** - 在提交给期刊或委员会之前验证学术写作
3. **质量保证** - 作为QA工作流程的一部分
4. **内容改写** - 将AI生成的内容改写为更自然的风格
5. **写作演变跟踪** - 监控写作风格随时间的变化

## 第一部分：24种AI写作模式检测

### 类别一：内容模式 (Content Patterns)

#### 1. 过度强调意义 (Symbolic Overemphasis)
**问题**：夸大事物的历史意义或象征价值
- 信号词：「标志着」「具有里程碑意义」「开创性的」「划时代的」
- 示例：「这标志着一个关键转折点」

**修复**：
- 删除夸张修饰词
- 用具体数据/事实替代
- 示例：→ 「这使效率提升了23%」或「成立于1989年」

#### 2. 知名度堆砌 (Name-Dropping)
**问题**：声称某事已被「广泛报道」或「获得国际认可」而无具体引用
- 信号词：「广受好评」「备受瞩目」「引发广泛关注」

**修复**：
- 删除无来源的知名度声明
- 或添加具体的引用来源

#### 3. -ing结尾的浅层分析 (Shallow -ing Analysis)
**问题**：用-ing动词结尾制造深度分析的假象
- 示例：「exploring」「demonstrating」「highlighting」「showcasing」

**修复**：
- 使用具体的动作动词
- 描述实际发生了什么

#### 4. 模糊归因 (Vague Attribution)
**问题**：引用不明确的来源
- 信号词：「专家表示」「研究表明」「据报道」「业内人士认为」

**修复**：
- 添加具体的专家/研究名称
- 或删除无法核实的声明

#### 5. 宣传性语言 (Propaganda Language)
**问题**：使用过于正面或推销性的形容词
- 信号词：「令人惊叹」「独一无二」「无与伦比」「revolutionary」「groundbreaking」

**修复**：
- 用中性描述替代
- 让事实自己说话

#### 6. 模板化结构 (Template Structure)
**问题**：「挑战与展望」「问题与机遇」等公式化章节
- 特征：每篇文章都有相似的开头/结尾结构

**修复**：
- 根据具体内容定制结构
- 避免千篇一律的模板

### 类别二：语言/语法模式 (Language/Grammar Patterns)

#### 7. AI词汇滥用 (AI Vocabulary Overuse)

**高风险词（英文）**：
| 词汇 | 替换建议 |
|------|----------|
| delve | examine, explore, look at |
| leverage | use, apply |
| utilize | use |
| robust | strong, reliable |
| comprehensive | complete, thorough |
| facilitate | enable, help, allow |
| innovative | new |
| cutting-edge | advanced, modern |

**高风险词（中文）**：
| 词汇 | 替换建议 |
|------|----------|
| 此外 | （删除）或用句号分开 |
| 值得注意的是 | （删除） |
| 首先...其次...最后 | 自然分段 |
| 综上所述 | （删除）或直接陈述结论 |
| 至关重要 | 重要、关键 |
| 不可或缺 | 必要、需要 |

**过度使用的过渡词**：
| 词汇 | 替换建议 |
|------|----------|
| furthermore | also, next, （删除） |
| moreover | additionally, （删除） |
| consequently | so, therefore |
| subsequently | then, later |
| nevertheless | but, however |

#### 8. 系动词回避 (Copula Avoidance)
**问题**：用复杂短语替代简单的「是」
- 示例：「serves as」→「is」，「functions as」→「is」

**修复**：
- 直接使用「是」「为」等简单系动词

#### 9. 否定排比 (Negation Parallelism)
**问题**：过度使用「不仅...而且...」「既...又...」结构
- 特征：几乎每段都有这种结构

**修复**：
- 用简单句替代
- 保持句式多样性

#### 10. 三连排比 (Rule of Three)
**问题**：强制使用三个并列项
- 示例：「快速、高效、可靠」

**修复**：
- 根据实际需要调整列表长度
- 不要为凑数而添加

#### 11. 同义词循环 (Synonym Cycling)
**问题**：在同一段落中用多个同义词描述同一事物
- 示例：「研究/调查/探索」「重要/关键/至关重要」

**修复**：
- 保持用词一致
- 选择最准确的一个词

#### 12. 虚假范围 (False Scope)
**问题**：使用「从A到B」结构夸大范围
- 示例：「从初学者到专家」「从小型企业到大型集团」

**修复**：
- 如果范围不重要，直接删除
- 如果重要，用具体数据支撑

### 类别三：风格模式 (Style Patterns)

#### 13. 破折号过度使用 (Dash Overuse)
**问题**：频繁使用——来补充说明
- 特征：每段都有多个破折号

**修复**：
- 改用括号或独立句子
- 减少使用频率

#### 14. 加粗过度 (Bold Overuse)
**问题**：过多的**加粗文本**
- 特征：几乎每段都有加粗

**修复**：
- 只在真正重要的地方使用
- 减少加粗频率

#### 15. 内联标题列表 (Inline Header Lists)
**问题**：「标题：内容」格式过度使用
- 示例：「优势：快速。劣势：成本高。」

**修复**：
- 改用自然段落叙述
- 或使用真正的列表格式

#### 16. 标题大写化 (Title Capitalization)
**问题**：不必要的首字母大写（英文）
- 示例：「The Importance Of Learning」

**修复**：
- 遵循正常的大写规则

#### 17. 表情符号装饰 (Emoji Decoration)
**问题**：在正式文档中使用表情符号
- 示例：「重要提示 🚀」

**修复**：
- 删除所有非必要的表情符号
- 学术/商务文档完全不使用

#### 18. 弯引号滥用 (Curly Quote Overuse)
**问题**：不一致的引号风格
- 特征：混用「」""''

**修复**：
- 保持引号风格一致
- 中文用「」，英文用" "

### 类别四：交流模式 (Communication Patterns)

#### 19. 机器人痕迹 (Chatbot Artifacts)
**问题**：保留了对话式AI的痕迹
- 信号词：「希望这对您有帮助」「如果您有任何问题」「当然可以！」

**修复**：
- 删除所有对话式表达
- 使用正式的书面语

#### 20. 知识截止声明 (Knowledge Cutoff Disclaimers)
**问题**：提及AI的知识限制
- 示例：「截至我所知」「根据我的训练数据」

**修复**：
- 完全删除这类声明

#### 21. 谄媚语气 (Obsequious Tone)
**问题**：过度恭维读者或话题
- 信号词：「这是一个很好的问题」「您提出了一个重要的观点」

**修复**：
- 直接回答问题
- 删除不必要的赞美

#### 22. 填充短语 (Filler Phrases)
**问题**：增加字数但不增加信息的短语
- 示例：「在当今社会」「众所周知」「不言而喻」

**修复**：
- 直接删除

#### 23. 过度限定 (Over-qualification)
**问题**：过多的条件限制和免责声明
- 示例：「在某种程度上」「可能」「或许」「取决于具体情况」

**修复**：
- 做出明确的陈述
- 只在真正需要时限定

#### 24. 泛泛正面结论 (Generic Positive Conclusions)
**问题**：以空洞的乐观结尾
- 示例：「未来可期」「前景广阔」「大有可为」

**修复**：
- 用具体的下一步行动替代
- 或直接结束，不需要总结

## 第二部分：检测评分系统

### 评分公式

**总体信心** = 各项加权平均：
- 语法完美度：20%
- 句子一致性：25%
- 段落结构：20%
- AI典型词：25%
- 标点模式：10%

每个指标评分 0-100，然后按权重合并。

### 信心级别

#### 低信心（0-30%）：可能是人类写作
**特征：**
- 自然的句子长度变化（5-40+ 个单词）
- 偶尔的语法不完美
- 真实的声音和自然的流畅性
- AI典型词极少（每1000个单词 0-2 个）

**行动：** ✅ 写作看起来真实，无需更改

#### 中等信心（30-70%）：可能有AI协助
**特征：**
- 句子结构有一定一致性
- AI典型模式与自然模式的混合
- 某些地方过于正式
- 每1000个单词 3-5 个AI典型词

**行动：** ⚠️ 审查标记的部分，选择性应用建议

#### 高信心（70-100%）：可能是AI生成的
**特征：**
- 所有指标过度一致
- 多个AI典型词簇
- 整体完美的语法和标点
- 每1000个单词 6 个以上AI典型词

**行动：** 🚫 需要重大修订，重写为真实的声音

## 第三部分：改写质量评估

### 五维度质量评分（1-10分）

1. **直接性 (Directness)**: 是否直奔主题，不绕弯子
2. **节奏感 (Rhythm)**: 句子长短是否有变化
3. **读者信任度 (Reader Trust)**: 是否让人觉得可信
4. **真实性 (Authenticity)**: 是否有人的声音和立场
5. **简洁性 (Conciseness)**: 是否去除了冗余

### 改写原则

> **「不仅要干净，更要鲜活」**

有效的去AI化不只是删除模式，还需要：
- 注入具体观点和立场
- 变化句式节奏
- 承认复杂性和不确定性
- 使用第一人称（当合适时）
- 允许适度不完美
- 用具体细节替代抽象概括

## 第五部分：集成配置

### 预提交钩子集成

```bash
# 在 .claude/settings.json 中配置
"gitPreCommit": {
  "command": "python3 hooks/pre-commit-ai-check.py",
  "enabled": true
}
```

**行为：**
- 在暂存的 `.md`, `.tex`, `.rst` 文件上运行
- 如果信心在 30-70% 之间发出警告
- 如果信心超过 70% 则阻止提交
- 用户可以通过 `git commit --no-verify` 覆盖

### QA集成

```yaml
# .ai-humanizer-assets/assets/config.yaml
qa_integration:
  enabled: true
  max_confidence_threshold: 0.40
  check_manuscripts: true
  check_documentation: true
  generate_detailed_reports: true
```

### 配置文件完整示例

```yaml
# AI-Check-Humanizer 配置

# 预提交钩子设置
pre_commit:
  enabled: true
  check_files: [".md", ".tex", ".rst", ".txt"]
  block_threshold: 0.70
  warn_threshold: 0.30
  exclude_patterns:
    - "*/examples/*"
    - "*/tests/*"

# 检测参数
detection:
  weights:
    grammar_perfection: 0.20
    sentence_uniformity: 0.25
    paragraph_structure: 0.20
    ai_word_frequency: 0.25
    punctuation_patterns: 0.10

  ai_words_per_1000_threshold: 3.0
  human_baseline_per_1000: 1.5

# 语言设置
language:
  primary: "zh"  # zh 或 en
  detect_both: true

# 改写设置
rewriting:
  preserve_meaning: true
  preserve_tone: true
  inject_personality: true

# 报告生成
reporting:
  default_format: "markdown"
  include_suggestions: true
  include_word_frequency: true
```

## 第六部分：使用示例

### 示例1：高信心检测与改写

**输入文本：**
```
此外，这项综合性研究深入探讨了用于促进创新方法实施的稳健方法论。
值得注意的是，该分析展示了利用尖端技术的重大发现。
综上所述，未来可期。
```

**检测报告：**
```
总体信心：85%（高 - 可能是AI生成）

发现的AI模式：
1. AI词汇滥用 - "此外"、"综合性"、"深入探讨"、"促进"、"稳健"、"值得注意的是"、"利用"、"尖端"、"综上所述"
2. 泛泛正面结论 - "未来可期"
3. 过度强调意义 - "重大发现"
```

**改写后：**
```
这项研究检验了实施新方法的技术。分析显示各指标有明显改善。
下一步是扩大测试范围。
```

**质量评分：**
```
直接性：    9/10
节奏感：    7/10
读者信任度：8/10
真实性：    8/10
简洁性：    9/10
总评：      8.2/10
```

### 示例2：中等信心检测

**输入文本：**
```
The experimental design followed standard protocols established in
previous work. We collected data from 150 participants over six months.
The results showed a significant main effect of condition (p < 0.001).
```

**检测报告：**
```
总体信心：35%（中等 - 可能有轻微AI协助）

发现的问题：
- 句子长度略微一致（11-15个单词）
- "significant" 在统计上下文中可接受
- 整体是自然的学术写作

建议：可选择性进行小修订，写作大体真实
```

### 示例3：低信心（人类写作）

**输入文本：**
```
好吧，这是我们发现的。效果很大——比预期大得多。实验组平均得分高了23%。
这不只是统计上显著，实际意义也很大。

我们还不太确定原因。可能是时机？也可能是说明更清楚了。需要做后续研究。
```

**检测报告：**
```
总体信心：12%（低 - 明显是人类写作）

人类写作指标：
- 自然的句子变化
- 非正式元素（"好吧"、"比预期大得多"）
- 不完整的想法和问题
- 没有AI典型词
- 整体真实的声音

建议：写作真实，无需更改
```

## 限制说明

### 此技能无法做到的事情

1. **不100%准确**
   - LLM 不断改进，模式随时间演变
   - 可能出现误报（非常正式的人类写作）
   - 可能出现漏报（经过大量编辑的AI文本）

2. **无法检测所有AI使用**
   - 编辑良好的AI文本可能通过
   - 改写工具可能逃避检测

3. **语言限制**
   - 主要针对中文和英文
   - 其他语言支持有限

### 与人类判断结合使用

此技能是工具，而不是人类判断的替代品：
- 将信心评分作为指导，而不是绝对真理
- 考虑上下文和领域特定规范
- 保持学术诚信标准
