# AI 检测与人性化改写技能

## 目的

本技能融合了 **AI检测** 与 **人性化改写** 两大功能：
1. **检测**：识别典型的 LLM 生成文本模式，评估AI痕迹程度
2. **改写**：提供详细的修复指南，让文本更自然、更有人味

> 核心理念：**不仅要干净，更要鲜活**

## 何时使用此技能

1. **预提交验证** — 在 git 提交之前自动检查文档
2. **手稿审查** — 在提交给期刊或委员会之前验证学术写作
3. **质量保证** — 作为QA工作流程的一部分
4. **内容改写** — 将AI生成的内容改写为更自然的风格
5. **写作演变跟踪** — 监控写作风格随时间的变化

## 24种AI写作模式（速查）

### 类别一：内容模式
1. **过度强调意义** — 「标志着」「里程碑意义」「划时代的」
2. **知名度堆砌** — 「广受好评」「备受瞩目」无具体引用
3. **-ing结尾浅层分析** — exploring/demonstrating/highlighting
4. **模糊归因** — 「专家表示」「研究表明」无具体来源
5. **宣传性语言** — 「令人惊叹」「revolutionary」「groundbreaking」
6. **模板化结构** — 「挑战与展望」「问题与机遇」公式化章节

### 类别二：语言/语法模式
7. **AI词汇滥用** — delve/leverage/utilize/robust/此外/值得注意的是
8. **系动词回避** — 「serves as」→「is」，「functions as」→「is」
9. **否定排比** — 过度使用「不仅...而且...」「既...又...」
10. **三连排比** — 强制使用三个并列项
11. **同义词循环** — 同段落用多个同义词描述同一事物
12. **虚假范围** — 「从初学者到专家」「从小型企业到大型集团」

### 类别三：风格模式
13. **破折号过度使用** — 频繁用——补充说明
14. **加粗过度** — 几乎每段都有**加粗文本**
15. **内联标题列表** — 「优势：内容。劣势：内容。」格式
16. **标题大写化** — 不必要的首字母大写（英文）
17. **表情符号装饰** — 正式文档中使用🚀等表情
18. **弯引号滥用** — 混用「」""''

### 类别四：交流模式
19. **机器人痕迹** — 「希望这对您有帮助」「当然可以！」
20. **知识截止声明** — 「截至我所知」「根据我的训练数据」
21. **谄媚语气** — 「这是一个很好的问题」
22. **填充短语** — 「在当今社会」「众所周知」「不言而喻」
23. **过度限定** — 「在某种程度上」「可能」「或许」过多
24. **泛泛正面结论** — 「未来可期」「前景广阔」「大有可为」

详细定义、修复指南和示例 → `references/patterns.md`

## 检测评分系统

### 评分公式

| 指标 | 权重 |
|------|------|
| 语法完美度 | 20% |
| 句子一致性 | 25% |
| 段落结构 | 20% |
| AI典型词频率 | 25% |
| 标点模式 | 10% |

### 信心级别

| 分数 | 级别 | 行动 |
|------|------|------|
| 0-30% | 低 — 可能是人类写作 | ✅ 无需更改 |
| 30-70% | 中 — 可能有AI协助 | ⚠️ 选择性应用建议 |
| 70-100% | 高 — 可能是AI生成 | 🚫 需要重大修订 |

## 句子级硬规则

以下句式**禁止使用**：

### 英文
- ❌ Wh- 疑问句开头（What/When/Where/Who/Why/How 引导的设问句）
- ❌ Em-dash（——）过多使用
- ❌ 强制三元素列表（只用两个或一个更自然）
- ❌ Lazy extremes（"always", "never", "everyone" 等绝对词）

### 中文
- ❌ 强制"首先...其次...最后"结构
- ❌ 过多的"——"破折号
- ❌ 四字成语堆砌（超过2个/段）

## 改写质量评估（五维度，1-10分）

总分 50分，**≥35分通过**，<35分需要修改

| 维度 | 分值 |
|------|------|
| 直接性 | 1-10 |
| 节奏感 | 1-10 |
| 读者信任度 | 1-10 |
| 真实性 | 1-10 |
| 简洁性 | 1-10 |
| **总分** | **50** |

### 改写原则

有效的去AI化不只是删除模式，还需要：
- 注入具体观点和立场（而非泛泛而谈）
- 使用主动语态（找到行动者并前置）
- 删除虚假主语（"the decision emerges" → "the team decided"）
- 变化句式节奏
- 承认复杂性和不确定性
- 使用第一人称（当合适时）
- 允许适度不完美
- 用具体细节替代抽象概括

## 集成配置

### 预提交钩子

```bash
# 在 .claude/settings.json 中配置
"gitPreCommit": {
  "command": "python3 hooks/pre-commit-ai-check.py",
  "enabled": true
}
```

行为：在暂存的 `.md`, `.tex`, `.rst` 文件上运行；30-70% 发出警告；>70% 阻止提交；可用 `git commit --no-verify` 覆盖。

### QA集成配置

```yaml
# .ai-humanizer-assets/assets/config.yaml
qa_integration:
  enabled: true
  max_confidence_threshold: 0.40
  check_manuscripts: true
  check_documentation: true
  generate_detailed_reports: true
```

完整配置 Schema → `references/patterns.md`

## 限制说明

1. **不100%准确** — 可能出现误报（非常正式的人类写作）或漏报（经过大量编辑的AI文本）
2. **无法检测所有AI使用** — 编辑良好的AI文本可能通过
3. **语言限制** — 主要针对中文和英文

将信心评分作为指导，而不是绝对真理。
